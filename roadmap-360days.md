# 운영 전산 360일 학습 로드맵

본 문서는 Linux 서버 운영, 인프라 관리, 장애 대응 역량을
체계적으로 기르기 위한 **360일(약 1년) 실습 중심 로드맵**입니다.  
하루 2~3시간 학습을 기준으로 설계되었습니다.

---

## 매일 공통 학습 루틴 (360일 공통)
- 이론 학습: 30분
- 실습: 60~90분
- 기록 정리(GitHub): 20~30분

모든 실습 결과는 Markdown 문서로 기록하여 GitHub에 업로드합니다.

## 1단계: Linux 기초 및 서버 운영 감각 (Day 1–60)
**목표:** Linux 서버 환경에 완전히 익숙해지기

### Day 1–10: 서버 설치 및 기본 명령어
- Ubuntu Server 설치 (여러 번 반복)
- SSH 설치 및 접속
- 기본 쉘 명령어(ls, cd, mkdir, rm, pwd)
- GUI 없는 서버 환경 이해

### Day 11–20: 사용자, 권한, 파일 시스템
- 사용자 및 그룹 관리
- sudo 개념 이해
- 파일 권한(chmod, chown)
- 디렉토리 구조(/etc, /var, /home)

### Day 21–30: 프로세스와 서비스
- 프로세스 개념 이해
- ps, top, kill 사용
- systemctl 기초
- 서비스 시작/중지/재시작

### Day 31–45: 자원 모니터링
- 디스크 사용량(df, du)
- 메모리 사용량(free)
- CPU 모니터링
- 로그 위치 및 기본 로그 읽기

### Day 46–60: 재설치 및 복습
- 서버 완전 재설치
- 사용자/SSH/서비스 재설정
- Linux 기초 전체 복습
- 문서 정리

**산출물**
- Linux 설치 기록 문서
- 기본 운영 정리 문서
